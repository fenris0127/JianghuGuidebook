## 1. 알려진 이슈 (Known Issues)

### 1.1 크리티컬 이슈 (해결 필요)

현재 크리티컬 이슈 없음 ✅

### 1.2 메이저 이슈

#### 이슈 #1: UI 프리팹 미구현
**설명**: UI 스크립트는 완성되었으나 실제 Unity 프리팹이 생성되지 않음

**영향**:
- 게임을 바로 실행할 수 없음
- Unity 에디터에서 수동 설정 필요

**해결 방법**:
1. Unity에서 Canvas 생성
2. 각 UI 스크립트에 맞는 UI 요소 생성
3. 프리팹으로 저장

**우선순위**: HIGH
**예상 작업 시간**: 2-4시간
**담당**: Phase 2

---

#### 이슈 #2: 상태 효과 시스템 미완성
**설명**: StatusEffect 기본 클래스만 구현됨, 실제 효과(중독, 화상 등) 미구현

**영향**:
- 마도수련자의 버프 행동이 작동하지 않음
- 일부 스킬 카드 효과 제한적

**해결 방법**:
1. 구체적인 상태 효과 클래스 구현 (Strength, Vulnerable, Weak 등)
2. 상태 효과 스택 시스템
3. 상태 효과 UI 표시

**우선순위**: HIGH
**예상 작업 시간**: 4-6시간
**담당**: Phase 2

---

#### 이슈 #3: 카드 타겟팅 시스템 단순화
**설명**: 현재 카드는 자동으로 첫 번째 적을 타겟팅

**영향**:
- 여러 적이 있을 때 선택 불가
- 전략성 제한

**해결 방법**:
1. 카드 드래그 시 타겟 선택 UI
2. 타겟 하이라이트 효과
3. 전체 공격/단일 타겟 구분

**우선순위**: MEDIUM
**예상 작업 시간**: 3-4시간
**담당**: Phase 2

---

## 3. 기술 부채 (Technical Debt)

### 3.1 코드 품질

#### 항목 #1: CombatUI의 책임 과다
**설명**: CombatUI가 너무 많은 책임을 가짐 (400+ 줄)

**개선 방안**:
- 카드 관리 → CardController로 분리
- 적 관리 → EnemyController로 분리
- 턴 관리 → TurnManager로 분리

**우선순위**: MEDIUM
**예상 작업 시간**: 4-6시간

---

#### 항목 #2: 하드코딩된 값들
**설명**: 일부 로직에 매직 넘버가 하드코딩됨

**예시**:
```csharp
// 나쁜 예
if (eventData.position.y > Screen.height * 0.4f)

// 좋은 예
if (eventData.position.y > Constants.CARD_PLAY_THRESHOLD)
```

**개선 방안**: Constants 클래스 확장 및 ScriptableObject 활용
**우선순위**: LOW

---

#### 항목 #3: 에러 처리 부족
**설명**: 일부 메서드에 예외 처리 없음

**개선 방안**: try-catch 블록 추가, null 체크 강화
**우선순위**: MEDIUM

---

### 3.2 아키텍처

#### 항목 #4: 이벤트 시스템 부재

## 4. 플레이테스트 피드백 (수집 예정)

### 4.1 피드백 수집 계획

**목표**:
- 2-3명의 외부 테스터
- 각 테스터당 30-60분 플레이
- 피드백 양식 작성

**수집 항목**:
1. 전반적인 재미도 (1-5)
2. 난이도 평가
3. UI/UX 개선점
4. 버그 발견
5. 추가 원하는 기능

**일정**: Phase 1 완료 후 1주일 이내

---

### 4.2 예상 피드백 (개발자 관점)

**긍정적 요소**:
- ✅ 카드 시스템이 직관적
- ✅ 전투가 전략적
- ✅ 무협 테마가 신선함

**개선 필요 요소**:
- ⚠️ 콘텐츠 부족 (카드 20장)
- ⚠️ 전투만 있고 메타 진행 없음
- ⚠️ 시각/청각 효과 부족
